## 백엔드(Django) 구현 지침
- 엔드포인트 및 권한 구현:StorePostListCreateView 및 StorePostDetailView를 구현하여 store/posts/ 및 store/posts/<int:pk>/ 엔드포인트를 활성화하세요.  IsAuthenticated와 IsBusinessUser 권한을 필수로 적용하세요.

- 조회 및 필터링 (GET):본인(author=request.user)이 작성한 board='local_shop' 게시글 목록만 반환해야 합니다.  쿼리 파라미터 ?ordering=latest|popular|viewed에 따라 정렬 조건이 바뀌도록 분기 처리 로직을 구현하세요.  상세 조회 시, 타인의 게시글 ID 접근을 원천 차단(403 Forbidden)하는 로직을 구현하세요.  

- 생성 (POST) 및 수정 (PUT):요청 데이터에 관계없이 백엔드 뷰단에서 board='local_shop', author=request.user를 강제로 주입하여 저장하세요.  기존 community/views.py에 정의된 save_images() 및 save_videos() 헬퍼 함수를 import하여 미디어 업로드를 처리하세요.  수정(PUT) 시 새로운 이미지가 들어오면, 명세서 요구사항에 따라 기존 이미지 레코드를 전체 삭제(delete())한 후 재업로드되도록 처리하세요.  

- 주소 데이터 유효성 검증 (핵심 예외 처리):우리 동네 가게 게시판에서 사업장 주소와 게시글 위치를 문자열로 매칭해야 하므로, 주소 입력 데이터의 공백 제거 및 규격 포맷 정제(Validation) 로직을 시리얼라이저나 뷰에 추가하세요.

## 프론트엔드(Vue 3) 구현 지침
- 가게 관리 CRUD UI 및 파일 업로드:게시글 목록(정렬 필터 포함), 작성, 수정, 삭제 폼 인터페이스를 구현하세요.  이미지 및 영상 다중 파일 첨부를 위한 multipart/form-data 전송 로직을 작성하세요.  

- 주소 API 연동 및 일치 보장 (핵심):주소 입력 시 오차 없는 매칭을 보장하기 위해, 프론트엔드에서 주소 검색 자동완성 API(예: 카카오 우편번호 서비스 등)를 연동하거나 사업자 등록 주소를 자동으로 불러와 바인딩하는 방식을 구현하여 임의의 텍스트 오타 입력을 방지하세요.
