## 백엔드(Django) 구현 지침
- 체험단 공고 CRUD 구현:ExperiencePostListCreateView 및 ExperiencePostDetailView를 구현하여 experience/posts/ 및 experience/posts/<int:pk>/ 엔드포인트를 활성화하세요.  게시글 저장 시 board='experience', category='recruit', author=request.user를 뷰단에서 강제로 고정하여 저장하세요.

- 신청자 목록 조회 API 구현:GET /api/business/experience/posts/{id}/applicants/ 엔드포인트를 구현하세요.  권한 검증: 요청을 보낸 유저가 해당 체험단 공고의 실제 작성자(post.author == request.user)인지 반드시 확인하고, 타인이 접근할 경우 403 Forbidden을 반환하세요. 

- 핵심: 제약 사항 및 예외 처리 로직 구현 (PRD & Spec 반영) ★:일반 사용자가 체험단 신청서(ExperienceApplication)를 제출하는 시점에 백엔드 밸리데이터(Serializer 또는 View)에서 다음 사항을 검증하세요 
- 권한 예외: 사업자 회원(user_type='business')이 체험단 신청을 시도하면 신청 불가 메시지와 함께 차단하세요.  
- 기간 예외: 현재 날짜가 모집 기간(recruit_end)을 초과한 체험단 공고에 신청을 시도하면 신청 불가 메시지와 함께 차단하세요.  
- 중복 예외: 해당 일반 회원이 이미 신청한 체험단 공고에 다시 재신청을 시도하면 중복 신청 불가 메시지와 함께 409 Conflict 에러를 반환하세요 (unique_together 위반 방지).

- 예외 상황 대응 안내: 백엔드에서 제약 사항 위반으로 에러(403, 409 등)를 반환할 경우, 이를 캐치하여 사용자에게 기획서에 명시된 안내 메시지("이미 신청한 체험단입니다", "모집 기간이 지났습니다" 등)를 얼럿(Alert)이나 모달로 친절하게 표시하세요.
